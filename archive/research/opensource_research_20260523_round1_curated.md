## Learned implementation notes

- Treat third-party registration projects as field/order references only; do not import their captcha/proxy/bulk-generation logic.
- For provider isolation, keep a separate step enum and detector per provider, then render only the selected provider flow in the extension.
- For SMS services, normalize every adapter to `list_numbers(provider,country)` and `fetch_messages(provider,number/message_url)` with `free/public/login_required/api_key_required` metadata.
- A provider is not usable until number listing, message page load, and code extraction all pass in one real probe.
- Store raw search results separately from curated candidates so future sweeps can dedupe by URL and repository full name.
